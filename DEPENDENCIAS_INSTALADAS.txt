# DEPENDENCIAS DEL PROYECTO - Repositorio ISER

## PROGRAMAS REQUERIDOS (Instalación del Sistema)

### 1. Node.js (>= 18.x)
- **Función**: Entorno de ejecución para JavaScript backend y frontend
- **Instalación**: `curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - && sudo apt-get install -y nodejs`
- **Alternativa**: Descargar desde https://nodejs.org/

### 2. npm (>= 10.x) o yarn
- **Función**: Gestor de paquetes de Node.js
- **Instalación**: Viene incluido con Node.js

### 3. MySQL (>= 8.0 o 5.7)
- **Función**: Base de datos relacional para almacenar usuarios, tesis y fragmentos
- **Instalación**: `sudo apt-get install mysql-server`
- **Alternativa**: Docker con `docker run --name mysql -e MYSQL_ROOT_PASSWORD=root -p 3306:3306 -d mysql:8.0`

| Paquete | Versión | Función |
|---------|---------|---------|
| openai | ^4.77.0 | SDK oficial de OpenAI para integración con GPT-4.1-mini |

### Dependencias de IA - Opción Gemini (Recomendado)
| Paquete | Función |
|---------|---------|
| @google/generative-ai | SDK oficial de Google Gemini AI |

Para usar Gemini en lugar de OpenAI, instalar:
```bash
npm install @google/generative-ai
```

Variables de entorno para Gemini en `.env`:
```
GEMINI_API_KEY=your-gemini-api-key-here
GEMINI_MODEL=gemini-1.5-flash
```

## COMANDOS DE INSTALACIÓN

### Backend
```bash
cd repositorio-backend
npm install
```

### Frontend
```bash
cd repositorio-frontend
npm install
```

### Migración de Base de Datos
```bash
# Ejecutar migración para crear tabla thesis_chunks (RAG)
mysql -u root -p < database/migrations/003_thesis_chunks.sql
```

## NOTAS

- El proyecto usa arquitectura limpia (controllers/services/routes)
- El sistema RAG está implementado con `openai` pero preparado para migrar a `@google/generative-ai` (Gemini)
- Docker Compose está disponible para despliegue con MySQL incluido
- Variables de entorno necesarias: `JWT_SECRET`, `OPENAI_API_KEY` (o `GEMINI_API_KEY`)
